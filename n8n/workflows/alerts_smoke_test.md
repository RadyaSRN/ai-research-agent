# alerts_smoke_test

**Trigger**: Schedule (не активный по умолчанию — запускается вручную)
**Active by default**: no

## Назначение

Ручной smoke-test для `alerts_error_handler`: намеренно бросает исключение, чтобы убедиться, что error workflow подключён и Telegram-алерт доходит до админа. Держится выключенным — активируется только на время проверки.

## Входы

- Нет.

## Выходы

- Искусственное падение воркфлоу → алерт в Telegram (через `alerts_error_handler`).

## Что делает по шагам

1. `Schedule Trigger` (с настроенным интервалом — активируется вручную).
2. `Throw error` — Code-нода, которая бросает `throw new Error("smoke test")`.

## Зависимости

- **Вызывает воркфлоу**: нет
- **Вызывается из**: нет
- **Credentials в n8n**: нет
- **Требует**: в `Workflow Settings → Error Workflow` у этого воркфлоу должен быть выбран `alerts_error_handler`.

## Примечания

Не забывать выключить после прогона, иначе cron продолжит периодически слать алерты.
